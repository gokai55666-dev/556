## 3. What to Do with Your `.md` / `.txt` Folders

To leverage your recent attempts (folders of `.md` and `.txt` files), the following usability-oriented setup helps:

1. **Project context folder**  
   Keep a root folder with short files like `project_description.md`, `constraints.md`, `guidelines.md`. These are read by the AI at session start.

2. **Workflow definition files**  
   Use individual `.md` files mapping to tasks or “skills” (e.g. `SKILL.md` in subfolders), each describing what needs to happen (action, input, output).

3. **Versioning / change log**  
   A file like `changelog.txt` or `versions.md` documenting what changed when and why — aids prompt tuning and traceability.

4. **Prompt library**  
   Folder of prompt templates: each has name, purpose, example input/output, constraints; e.g. `prompt_templates/generate_summary.md`.

5. **Task examples / negative examples**  
   Include good outputs and “what not to do” examples to constrain and guide the AI behavior.

---

## 4. Usability Considerations

- **Readability:** files are short, plain, structured (headings, bullet lists).  
- **Discoverability:** predictable folder structure and naming; documentation to explain what folder is used for what.  
- **Modifiability:** easy to update files without breaking workflows / agents.  
- **Traceability:** every prompt version linked with examples and outcomes (so you can see what works).

## 5. Prompt You Can Use to Build the Workflow “As Intended”

Here’s a refined prompt you can use with an LLM to generate or refine your AI workflow system, based on your `.md` / `.txt` folder setup. You should run this once you have your folders in place.

---

### Workflow-Building Prompt

```
# Context
I have a project folder structured with multiple `.md` and `.txt` files.  
These include:
- `project_description.md`
- `constraints.md`
- A `skills/` folder containing `SKILL.md` files defining tasks or actions
- A `prompt_templates/` folder with prompt templates, good & bad examples
- A `changelog.md` or `versions.txt` for version tracking

# Task
Help me produce a fully specified AI workflow architecture and implementation plan that uses these files. The AI should use them to:

1. Load context at session start  
2. Interpret the `skills` folder tasks into executable actions  
3. Use prompt templates for each task  
4. Enforce constraints and guidelines from `constraints.md`  
5. Record changes in versioning / changelog

# Requirements / Constraints:

- Use modular tasks: each SKILL.md defines a self-contained unit with input, output, dependencies  
- Ensure each prompt template includes: purpose, example, input format, output format, tone/style constraints  
- Provide mechanism to validate outputs against constraints and “what not to do” examples  
- Design a version control or change-logging workflow: how updates get reviewed, tracked, applied  
- Suggested file/folder naming conventions  
- Include suggestions for automations (e.g. hooks, startup reads, agent interactions)

# Output format

Provide output with these sections:

## Workflow Architecture  
Explain the components (context loader, task executor, prompt template manager, constraint enforcer, version tracker), and how they interact.

## File Structure Example  
List a sample folder structure with file names and their purpose.

## Template SKILL.md format  
Provide a skeleton SKILL.md file with sections and metadata.

## Template prompt template format  
Provide a skeleton template file for prompt templates including examples and constraints.

## Governance & Versioning Plan  
Explain how the system will evolve safely with version control, review steps, and tracking.

## Implementation Steps  
Step by step plan to build this workflow system from your current folder setup.

```
